# Design Questionnaire — Guide for Implementation

## Overview

This guide helps you effectively distribute the design questionnaire to clients, collect feedback, and synthesize insights for actionable improvements.

---

## Distribution Tips

### Format Options

1. **PDF Form** — Print/email as a formal document
2. **Google Form** — Create an online survey for easy response collection
3. **Typeform/Qualtrics** — Professional survey platform with analytics
4. **Email** — Send as markdown or formatted document for direct responses

### Distribution Strategy

- **Timing:** Send 1-2 weeks after site launch when client has time to review thoroughly
- **Follow-up:** No follow-up needed; set a deadline (typically 5-7 business days)
- **Incentive:** Emphasize that detailed feedback directly influences improvements
- **Instructions:** "Please take 15-20 minutes to complete. Honest feedback appreciated."

## Key Sections Explained

### Visual Identity (Sections 1-3)

**What to look for:** Emotional reactions to colors, typography, and overall feel.

- Strong agreement on color palette = good brand fit
- Disagreement = consider alternatives or gather "why" details
- Watch for pattern: if many prefer different colors, explore alternatives

### Homepage & Hero (Sections 2)

**What to look for:** Whether the first impression works.

- If engagement scores low = hero needs refresh
- If flow scores low = consider reorganizing sections

### Board of Directors (Section 4)

**What to look for:** Premium feel and trust-building effectiveness.

- High praise = keep as differentiator
- Animation concerns = may need to reduce or adjust speeds
- Low engagement = might move section or redesign

### Content & Messaging (Section 7)

**What to look for:** Clarity and resonance with target audience.

- Unclear tagline = work with client on value prop
- Long-form sections marked "too long" = need to condense

### Trust & Credibility (Section 8)

**What to look for:** What signals are working and what gaps exist.

- Ask follow-ups about gaps identified
- Prioritize credibility improvements if scores are low

## Data Analysis Framework

### Step 1: Quantitative Scoring

Create a simple scoring system for rating questions:

- Excellent/Strongly Agree = 4 points
- Good/Agree = 3 points
- Fair/Neutral = 2 points
- Poor/Disagree = 1 point
- Strongly Disagree = 0 points

Calculate averages by section to identify strengths and weaknesses.

**Example:**

```
Visual Identity — Average: 3.2/4 (Good, minor improvements)
Content & Messaging — Average: 2.8/4 (Fair, needs work)
Overall Impression — Average: 3.5/4 (Good)
```

### Step 2: Qualitative Themes

Group open-ended responses by theme:

**Common Patterns:**

- Repeat mentions of specific words/concerns
- Similar suggestions from multiple respondents
- Conflicting feedback (some want more, some want less)

**Document:**

- Most requested feature
- Most common concern
- Most mentioned strength
- Most controversial element

### Step 3: Priority Matrix

Rank feedback by:

1. **Impact — How much would fixing this improve the site?** (High/Medium/Low)
2. **Effort — How much work is required?** (Easy/Medium/Complex)
3. **Priority — Urgency based on client goals**

**Prioritization Grid:**

```
High Impact + Easy Effort = Do First (Quick Wins)
High Impact + Complex = Plan & Schedule
Low Impact + Easy = Nice to Have
Low Impact + Complex = Deprioritize
```

## Running an Effective Feedback Call

After collecting responses, schedule a 30-45 minute call with the client.

### Call Agenda

1. **Thank you** (2 min) — Appreciate detailed feedback
2. **Quantitative overview** (5 min) — Share scores and general sentiment
3. **Top wins to celebrate** (5 min) — What's working well
4. **Key concerns** (10 min) — Top 3 areas for improvement + discussion
5. **Quick wins** (5 min) — What we can fix in 1-2 weeks
6. **Roadmap** (5 min) — Timeline for larger changes
7. **Next steps** (3 min) — Due dates, next review, implementation plan

### Questions to Ask During Call

- "When you said [concern], what specifically bothered you?"
- "Of the improvements we discussed, which 3 are most important?"
- "What would represent success in your mind?"
- "Are there any industry benchmarks or competitors we should review?"

## Feedback Response Strategy

### If feedback is overwhelmingly positive:

✓ Celebrate success
✓ Maintain design direction
✓ Focus on continuous improvement
✓ Plan next review in 3-6 months

### If feedback is mixed:

✓ Identify patterns (which users loved/disliked what?)
✓ Ask clarifying questions
✓ Prioritize high-impact improvements
✓ Design A/B test variants if consensus is unclear

### If feedback is critical:

✓ Don't get defensive
✓ Ask for specific examples
✓ Understand underlying needs vs. surface complaints
✓ Propose solutions to address core concerns
✓ Plan more frequent review cycles

## Follow-up Actions

### Post-Feedback Checklist

- [ ] Thank all respondents
- [ ] Document all feedback in shared system
- [ ] Create priority matrix
- [ ] Schedule implementation review with stakeholders
- [ ] Communication plan for changes (inform client of timeline)
- [ ] Track implementation progress
- [ ] Schedule next feedback cycle (3-6 months)
- [ ] A/B test any major redesign changes
- [ ] Document learnings for future projects

## Sample Feedback Scenarios & Responses

### Scenario 1: "The hero slider is too busy"

**Analysis:** Animation/movement concerns
**Response Options:**

1. Reduce animation speed
2. Remove auto-play, let users control clicks
3. Simplify slide content
4. Replace with static hero

**Follow-up:** "What specifically felt overwhelming?"

### Scenario 2: "I don't understand what you do"

**Analysis:** Messaging/clarity issue
**Response Options:**

1. Revise headline/tagline for clarity
2. Add value proposition section above the fold
3. Simplify navigation/information hierarchy
4. Add icons/infographics to explain services

**Follow-up:** "Which section was unclear?"

### Scenario 3: "The board section is confusing"

**Analysis:** Interaction/UX concern
**Response Options:**

1. Simplify the layout
2. Add instructions or labels
3. Change interaction method (tabs vs. carousel)
4. Add context/tooltips

**Follow-up:** "What specifically was confusing?"

## Metrics to Track

Post-launch, measure if improvements move these metrics:

- **Engagement:** Time on page, scroll depth, click-through rates
- **Conversion:** Contact form submissions, newsletter signups
- **Trust:** Bounce rate, return visitors, social shares
- **Accessibility:** Accessibility audit scores
- **Performance:** Page load time, Core Web Vitals
- **User feedback:** NPS, CSAT scores from follow-up surveys

## Tips for Success

1. **Be specific** — Avoid vague feedback. Drill down into concrete examples.
2. **Look for patterns** — Single complaints matter less than repeated issues.
3. **Listen for underlying needs** — Sometimes what's said isn't what's meant.
4. **Balance feedback weights** — Client feedback > secondary stakeholders > best practices.
5. **Set expectations** — Not all feedback will be acted on; prioritize strategically.
6. **Close the loop** — Tell client which feedback was implemented and why others were deprioritized.
7. **Continuous cycle** — Feedback is ongoing, not a one-time event.
